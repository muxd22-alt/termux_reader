Introducing GitHub Copilot Workspace

GitHub StaffApril 29, 2024

Today we are excited to announce GitHub Copilot Workspace, a new developer environment that reimagines how software is built from idea to implementation.

Copilot Workspace starts with a natural language description of a task and builds a complete plan for implementing it. The developer can then review, modify, and execute the plan within the workspace.

From Issue to Pull Request

The typical workflow begins when a developer opens a GitHub issue. Copilot Workspace analyzes the issue description, the repository structure, and relevant code to propose a step-by-step implementation plan.

Each step in the plan includes specific file modifications, new file creation, or deletion suggestions. The developer has full control to accept, reject, or modify any suggested change before it is applied.

Integrated Development Flow

What makes Copilot Workspace unique is its deep integration with the GitHub platform. Changes are automatically committed to a branch, CI/CD pipelines run against the proposed changes, and pull requests are created with full context.

Early Access

Copilot Workspace is now available in technical preview for GitHub Copilot Enterprise customers. We are collecting feedback to refine the experience before a broader launch later this year.

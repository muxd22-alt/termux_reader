Goの並行処理パターン

go_dev2024-04-01

Go言語の最大の特徴の一つは、goroutineとchannelによる並行処理モデルです。この記事では、実務でよく使われる並行処理パターンを紹介します。

Fan-Out / Fan-In パターン

Fan-Outは一つのチャネルからの入力を複数のgoroutineに分散させるパターンです。Fan-Inはその逆で、複数のチャネルからの出力を一つのチャネルに集約します。このパターンはCPUバウンドな処理の並列化に特に有効です。

Worker Pool パターン

Worker Poolパターンは、固定数のgoroutineが共有のジョブキューからタスクを取り出して処理するパターンです。リソースの使用量を制御しながら並行処理を行うことができます。

func worker(id int, jobs <-chan int, results chan<- int) {
    for j := range jobs {
        results <- j * 2
    }
}

Context によるキャンセレーション

Go 1.7で導入されたcontextパッケージは、goroutine間のキャンセレーション伝播とデッドライン管理を提供します。長時間実行されるgoroutineは必ずcontextを受け取り、Done()チャネルを監視すべきです。

これらのパターンを適切に組み合わせることで、堅牢でスケーラブルな並行処理システムを構築できます。重要なのは、共有メモリではなくチャネルによるコミュニケーションを優先することです。

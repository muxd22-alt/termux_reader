Authentication

Overview

Our API supports two authentication methods: OAuth 2.0 for user-level access and API keys for server-to-server communication. All API requests must be authenticated.

OAuth 2.0

OAuth 2.0 is the recommended authentication method for applications that access resources on behalf of users. We support the Authorization Code flow and the Client Credentials flow.

Authorization Code Flow

The authorization code flow is used when your application needs to access the API on behalf of a specific user. The flow involves redirecting the user to our authorization server, obtaining an authorization code, and exchanging it for an access token.

POST /oauth/token
Content-Type: application/x-www-form-urlencoded

grant_type=authorization_code&
code=AUTH_CODE&
redirect_uri=https://yourapp.com/callback&
client_id=YOUR_CLIENT_ID&
client_secret=YOUR_CLIENT_SECRET

API Keys

API keys provide a simpler authentication mechanism for server-to-server integrations. Include your API key in the Authorization header of every request.

curl -H "Authorization: Bearer YOUR_API_KEY" \
  https://api.example.com/v1/resources

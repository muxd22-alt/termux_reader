React Server Componentsの実践的な使い方

Taro Yamada2024-03-15

React Server Components（RSC）は、Reactのアーキテクチャに大きな変革をもたらしました。この記事では、Next.js App Routerを使った実践的なRSCの活用方法を解説します。

Server Components vs Client Components

Server Componentsはサーバー側でのみレンダリングされ、JavaScriptバンドルに含まれません。一方、Client Componentsはブラウザで実行され、インタラクティビティを提供します。use clientディレクティブで明示的に区別します。

// Server Component (default in App Router)
async function UserProfile({ userId }: { userId: string }) {
  const user = await db.user.findUnique({ where: { id: userId } })
  return <div>{user.name}</div>
}

// Client Component
'use client'
function LikeButton() {
  const [liked, setLiked] = useState(false)
  return <button onClick={() => setLiked(!liked)}>Like</button>
}

Data Fetching パターン

RSCの最大のメリットは、コンポーネント内で直接データベースクエリやAPI呼び出しを行えることです。これにより、waterfall問題を解消し、必要なデータを効率的に取得できます。

Streaming と Suspense

Server ComponentsはReact Suspenseと組み合わせることで、段階的なストリーミングレンダリングを実現できます。遅いデータフェッチがあっても、準備ができた部分から順にクライアントに送信されます。

RSCはまだ進化の途中ですが、適切に使えばユーザー体験とDeveloper Experienceの両方を大幅に改善できるパラダイムです。
